Conclusion: MD5 is much faster to compute, so the same dictionary attack finds the password sooner. SHA-256 is slower per hash, making brute-force and dictionary attacks more expensive. For passwords, use a dedicated slow hash (e.g. bcrypt, Argon2) with salt.
